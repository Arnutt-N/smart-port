**Key functions:**

`fetchData()` -- for sub-tab level pages:
- Set `loading = true`, `error = null`
- Call `fetchByLevel(activeSubTab.value, { search: searchQuery.value, limit: pagination.value.limit, offset: pagination.value.offset })`
- On success: update `rows`, `summary`, `pagination`
- On error: set `error` to error message
- Set `loading = false`

`fetchOverviewData()` -- for overview page (per D-08):
- Set `overviewLoading = true`, `overviewError = null`
- Use `Promise.allSettled()` to fetch all 5 levels: O2, O3, K2, K3, K4 (Pitfall 2)
- Process results:
  - `generalTotal` = sum of O2.summary.total + O3.summary.total (from fulfilled results)
  - `academicTotal` = sum of K2 + K3 + K4 summary.total
  - `qualifiedTotal` = sum of all summary.qualified
  - `notYetTotal` = sum of all summary.not_yet
  - `checkDataTotal` = sum of all summary.check_data
  - `top5` = concatenate all data arrays, sort by remainingDays ascending (null last), take first 5
- Store in `overviewData` ref
- Set `overviewLoading = false`

`onSearchInput()` -- debounced search (Pitfall 3):
- Clear existing timeout: `clearTimeout(searchTimeout)`
- Set new timeout: `searchTimeout = setTimeout(() => { pagination.value.offset = 0; fetchData() }, 300)`

**Watchers:**
- `watch(() => props.section, ...)` -- when section changes: if overview, call fetchOverviewData(). If placeholder, do nothing. Otherwise, set activeSubTab to first sub-tab level, reset search/offset, fetch.
- `watch(activeSubTab, ...)` -- reset offset to 0 (Pitfall 4), reset searchQuery, call fetchData(). Only if not overview/placeholder.

**onMounted:** trigger initial load based on current section.
