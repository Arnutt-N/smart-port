**getEquivalenceList(PDO $pdo): void** (per D-02, D-03)

- Include all columns: equivalence_id, personnel_id, actual_position, equivalent_type, request_start_date, request_end_date, request_total_days, approved_start_date, approved_end_date, approved_total_days, approval_status, approved_by, approval_order_ref, created_at
- ORDER BY pe.created_at DESC, LIMIT/OFFSET
- Count query for pagination
- Loop adding Thai dates: request_start_date_thai, request_end_date_thai, approved_start_date_thai, approved_end_date_thai
- Response: `{ "success": true, "data": [...], "pagination": { ... } }`

**getEquivalenceDetail(PDO $pdo, int $id): void**

- Response: `{ "success": true, "data": {record} }`

**createEquivalence(PDO $pdo): void** (per D-14)
- Read JSON body
- Required fields: `['personnel_id', 'actual_position', 'equivalent_type', 'request_start_date', 'request_end_date']`
- Validation loop with Thai error: `"กรุณาระบุ {$field}"`
- Compute request_total_days: `(new DateTime($data['request_end_date']))->diff(new DateTime($data['request_start_date']))->days + 1`
- INSERT with approval_status hardcoded to 'PENDING' (per D-14 — client CANNOT set status on creation)
- Columns: personnel_id, actual_position, equivalent_type, request_start_date, request_end_date, request_total_days, approval_status (always 'PENDING')
- Also insert approval_order_ref if provided
- Return 201: `{ "success": true, "equivalence_id": N }` (per D-16)

**updateEquivalence(PDO $pdo, int $id): void** (per D-11, D-12, D-13)
- Read JSON body
- Fetch current record: `SELECT approval_status, request_start_date, request_end_date FROM position_equivalence WHERE equivalence_id = ?`
- 404 if not found: `'ไม่พบรายการเทียบตำแหน่ง'`

**Status transition enforcement (per D-11):**
- If `$data['approval_status']` is set:
  - Define valid transitions: `$validTransitions = ['PENDING' => ['APPROVED', 'REJECTED']];`
  - Check: `if (!in_array($newStatus, $validTransitions[$currentStatus] ?? []))` return 400 with `"ไม่สามารถเปลี่ยนสถานะจาก {$currentStatus} เป็น {$newStatus}"`

**On APPROVED (per D-12):**
- Require `approved_start_date` and `approved_end_date` in request body. Return 400 with `'กรุณาระบุ approved_start_date และ approved_end_date'` if missing.
- Compute: `$approvedTotalDays = (new DateTime($data['approved_end_date']))->diff(new DateTime($data['approved_start_date']))->days + 1`
- Extract user_id from JWT token: `$token = getAuthHeader(); $payload = json_decode(base64_decode(explode('.', $token)[1]), true); $approvedBy = $payload['data']['user_id'] ?? null;`
- UPDATE SET: approval_status='APPROVED', approved_start_date, approved_end_date, approved_total_days, approved_by, approval_order_ref (if provided)

**On REJECTED (per D-13):**
- UPDATE SET: approval_status='REJECTED', approved_start_date=NULL, approved_end_date=NULL, approved_total_days=NULL

**Non-status updates (if no approval_status in request):**
- Allowed fields: `['actual_position', 'equivalent_type', 'request_start_date', 'request_end_date', 'approval_order_ref']`
- If request_start_date or request_end_date changes, recompute request_total_days
- Only allow non-status updates when current status is PENDING. If APPROVED or REJECTED, return 400 with `'ไม่สามารถแก้ไขรายการที่อนุมัติหรือปฏิเสธแล้ว'`

- Return `{ "success": true }`

**deleteEquivalence(PDO $pdo, int $id): void**
- DELETE FROM position_equivalence WHERE equivalence_id = ?
- rowCount() check, 404 with `'ไม่พบรายการเทียบตำแหน่ง'`
- Return `{ "success": true }`
