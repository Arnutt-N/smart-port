CREATE TABLE position_equivalence (

**Main function:** `handleEquivalence(PDO $pdo, string $method, array $path): void` with switch on $method. Note: no DELETE for equivalence — only GET, POST, PUT per PE-01 (GET list, POST request, PUT approve/reject).

**Sub-functions to implement:**

1. `getEquivalenceList(PDO $pdo): void`
   - Read `$_GET['personnel_id']`, `$_GET['limit']` (default 20), `$_GET['offset']` (default 0) per D-02, D-03
   - SQL: `SELECT pe.*, CONCAT(p.first_name, ' ', p.last_name) AS full_name, CONCAT(u.first_name, ' ', u.last_name) AS approved_by_name FROM position_equivalence pe LEFT JOIN personnel p ON pe.personnel_id = p.personnel_id LEFT JOIN users u ON pe.approved_by = u.user_id`
   - Add `WHERE pe.personnel_id = ?` if personnel_id provided
   - `ORDER BY pe.created_at DESC LIMIT {$limit} OFFSET {$offset}`
   - Thai date formatting: loop rows, add `request_start_date_thai`, `request_end_date_thai`, `approved_start_date_thai`, `approved_end_date_thai` via `formatThaiDate()`
   - Count query for pagination total
   - Response: `{ "success": true, "data": [...], "pagination": { "total": N, "limit": N, "offset": N, "has_more": bool } }`

2. `getEquivalenceDetail(PDO $pdo, int $id): void`
   - Same SQL as list but `WHERE pe.equivalence_id = ?`
   - 404 if not found: `{ "error": "ไม่พบรายการเทียบตำแหน่ง" }` per D-15
   - Add Thai dates for all date fields
   - Response: `{ "success": true, "data": {...} }`

3. `createEquivalence(PDO $pdo): void`
   - Parse JSON body
   - Required fields: `['personnel_id', 'actual_position', 'equivalent_type']` — Thai error per D-15
   - **approval_status is ALWAYS 'PENDING' on creation** — ignore any client-submitted status (D-14)
   - **Server-side computation:** If both `request_start_date` and `request_end_date` provided, compute `$requestTotalDays = (new DateTime($data['request_end_date']))->diff(new DateTime($data['request_start_date']))->days + 1` (DATEDIFF+1, per research Open Question 4)
   - INSERT columns: `personnel_id, actual_position, equivalent_type, request_start_date, request_end_date, request_total_days, approval_order_ref, approval_status`
   - approval_status value is hardcoded `'PENDING'`
   - Response: `http_response_code(201); json_encode(['success' => true, 'equivalence_id' => intval($pdo->lastInsertId())])` per D-16

4. `updateEquivalence(PDO $pdo, int $id): void`
   - **This is the most complex handler — it handles both regular field updates AND approval status transitions.**
   - Parse JSON body
   - Fetch current record: `SELECT * FROM position_equivalence WHERE equivalence_id = ?`
   - 404 if not found per D-15
   - **Check for approval_status change (D-11, D-12, D-13):**
     - `$newStatus = $data['approval_status'] ?? null`
     - If `$newStatus` is set, validate transition:
       - `$validTransitions = ['PENDING' => ['APPROVED', 'REJECTED']]`
       - If `$current['approval_status']` is not in `$validTransitions` keys, or `$newStatus` is not in `$validTransitions[$current['approval_status']]`, return 400: `"ไม่สามารถเปลี่ยนสถานะจาก {$current['approval_status']} เป็น {$newStatus}"` per D-15
     - **If APPROVED (D-12):**
       - Require `$data['approved_start_date']` and `$data['approved_end_date']` — 400 if missing: `"กรุณาระบุวันที่เริ่มต้นและสิ้นสุดที่อนุมัติ"`
       - Compute: `$approvedTotalDays = (new DateTime($data['approved_end_date']))->diff(new DateTime($data['approved_start_date']))->days + 1`
       - Extract JWT user_id: `$token = getAuthHeader(); $payload = validateJWT($token); $userId = $payload['user_id'] ?? null;`
       - UPDATE: `SET approval_status = 'APPROVED', approved_start_date = ?, approved_end_date = ?, approved_total_days = ?, approved_by = ? WHERE equivalence_id = ?`
       - Execute with params: `[$data['approved_start_date'], $data['approved_end_date'], $approvedTotalDays, $userId, $id]`
     - **If REJECTED (D-13):**
       - UPDATE: `SET approval_status = 'REJECTED', approved_start_date = NULL, approved_end_date = NULL, approved_total_days = NULL WHERE equivalence_id = ?`
       - Execute with params: `[$id]`
     - Response: `{ "success": true }`
   - **If no status change (regular field update):**
     - Allowed fields whitelist: `['actual_position', 'equivalent_type', 'request_start_date', 'request_end_date', 'approval_order_ref']`
     - Build dynamic SET clause
     - If request dates changed, recompute `request_total_days` and add to SET
     - Response: `{ "success": true }`
