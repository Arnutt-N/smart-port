**getSupportiveList(PDO $pdo): void** (per D-02, D-03)
- Read `$_GET['personnel_id']`, `$_GET['limit']` (default 20), `$_GET['offset']` (default 0)
- Build WHERE clause: `WHERE se.personnel_id = ?` if personnel_id provided
- SELECT query joining personnel for name: `SELECT se.*, CONCAT(p.first_name, ' ', p.last_name) AS full_name FROM supportive_experience se LEFT JOIN personnel p ON se.personnel_id = p.personnel_id`
- Include all columns: supportive_id, personnel_id, job_series_name, start_date, end_date, total_days, ratio_percent, effective_days, net_end_date, net_years, net_months, net_day_remainder, description, created_at
- ORDER BY se.start_date DESC, LIMIT/OFFSET
- Count query for pagination total
- Loop through rows adding Thai date suffixed fields: `start_date_thai`, `end_date_thai`, `net_end_date_thai` using `formatThaiDate()`
- Response format (per D-16): `{ "success": true, "data": [...], "pagination": { "total": N, "limit": N, "offset": N, "has_more": bool } }`

**getSupportiveDetail(PDO $pdo, int $id): void**
- SELECT with JOIN to personnel for full_name, same columns as list
- WHERE `se.supportive_id = ?`
- 404 if not found: `['error' => 'ไม่พบรายการนับเกื้อกูล']` (per D-15)
- Add Thai date fields
- Response: `{ "success": true, "data": {record} }`

**createSupportive(PDO $pdo): void** (per D-04, D-05, D-06, D-07)
- Read JSON body via `json_decode(file_get_contents('php://input'), true)`
- Required fields: `['personnel_id', 'job_series_name', 'start_date', 'end_date']`
- Validation loop returning 400 with Thai message: `"กรุณาระบุ {$field}"` (per D-15)
- Server-side date computation (per D-05): `$startDate = new DateTime($data['start_date']); $endDate = new DateTime($data['end_date']); $totalDays = $endDate->diff($startDate)->days + 1;`
- Ratio lookup (per D-04): If `$data['primary_series_name']` is set, query `SELECT ratio_percent FROM supportive_job_series WHERE primary_series_name = ? AND supportive_series_name = ? AND is_active = 1 LIMIT 1`. Use `$data['primary_series_name']` as primary, `$data['job_series_name']` as supportive. Default `$ratioPercent = 100` if no match.
- Compute effective_days (per D-06): `$effectiveDays = $totalDays * $ratioPercent / 100;`
- Compute net_* fields (per D-07):
  ```
  $netYears = floor($effectiveDays / 365);
  $netMonths = floor(($effectiveDays % 365) / 30);
  $netDayRemainder = intval($effectiveDays) % 365 % 30;
  $netEndDate = (clone $startDate)->modify('+' . intval($effectiveDays) . ' days')->format('Y-m-d');
  ```
- INSERT into supportive_experience with columns: personnel_id, job_series_name, start_date, end_date, total_days, ratio_percent, effective_days, net_end_date, net_years, net_months, net_day_remainder, description
- Return 201 with `{ "success": true, "supportive_id": N }` (per D-16)

**updateSupportive(PDO $pdo, int $id): void**
- Read JSON body
- First verify record exists: `SELECT supportive_id FROM supportive_experience WHERE supportive_id = ?`. Return 404 with `'ไม่พบรายการนับเกื้อกูล'` if not found.
- Allowed fields for direct update: `['job_series_name', 'start_date', 'end_date', 'description']`
- Build SET clause using allowed-fields whitelist pattern from probation.php
- If start_date or end_date is being updated, fetch the current record's dates, merge with new values, and recompute: total_days, effective_days (look up ratio again using primary_series_name from request or current job_series_name), net_end_date, net_years, net_months, net_day_remainder. Add these computed fields to the SET clause.
- If job_series_name changes and primary_series_name is provided, re-lookup ratio and recompute effective_days and net_* fields.
- Execute UPDATE with prepared statement
- Return `{ "success": true }`

**deleteSupportive(PDO $pdo, int $id): void**
- DELETE FROM supportive_experience WHERE supportive_id = ?
- Check rowCount(), return 404 with `'ไม่พบรายการนับเกื้อกูล'` if 0
- Return `{ "success": true }`
