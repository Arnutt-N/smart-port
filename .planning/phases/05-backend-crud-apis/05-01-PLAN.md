   - Parse JSON body from `php://input`
   - Required fields: `['personnel_id', 'job_series_name', 'start_date', 'end_date']` — loop with Thai error per D-15
   - **Server-side computation (per D-05, D-06, D-07, SE-04):**
     - `$totalDays = $endDate->diff($startDate)->days + 1` (inclusive, D-05)

AND is_active = 1 LIMIT 1` — value for `supportive_series_name` comes from `$data['job_series_name']` (CRITICAL: column name mismatch between tables). Default ratio 100 if no match per D-04.
     - `$effectiveDays = $totalDays * $ratioPercent / 100` (D-06)
     - `$flooredEffective = intval(floor($effectiveDays))` — use floored value for net_* since effective_days is DECIMAL(10,2)
     - `$netYears = intval(floor($flooredEffective / 365))` (D-07)
     - `$netMonths = intval(floor(($flooredEffective % 365) / 30))` (D-07)
     - `$netDayRemainder = intval($flooredEffective % 365 % 30)` (D-07)
     - `$netEndDate = clone $startDate; $netEndDate->modify("+{$flooredEffective} days"); $netEndDateStr = $netEndDate->format('Y-m-d')`
   - INSERT INTO supportive_experience columns: `personnel_id, job_series_name, start_date, end_date, total_days, ratio_percent, effective_days, net_end_date, net_years, net_months, net_day_remainder, description`
   - Response: `http_response_code(201); json_encode(['success' => true, 'supportive_id' => intval($pdo->lastInsertId())])` per D-16

   - If `start_date` or `end_date` is in the update, fetch the other from DB if not provided, then RECOMPUTE all server-side fields (total_days, effective_days, net_*) using same logic as create
   - If `job_series_name` or `primary_series_name` changes, also recompute ratio lookup
   - Build dynamic SET clause from allowed fields + computed fields
   - Response: `{ "success": true }` per D-16
