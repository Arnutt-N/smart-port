    **Method: computeForLevel(string $targetLevel, ?string $search, int $limit, int $offset): array**

    This is the core method implementing CL-01, CL-04, CL-05:

    1. Query promotion_criteria to get DISTINCT source_level_code(s) for the target level:
       `SELECT DISTINCT source_level_code FROM promotion_criteria WHERE target_level_code = ? AND is_active = 1`

    2. Build the main qualification query using the SQL from Research (Pattern 3 / Code Examples):
       - JOIN personnel p with position pos (LEFT JOIN on current_position_id) and organization o (LEFT JOIN on current_org_id)
       - LEFT JOIN promotion_criteria pc ON pc.target_level_code = :targetLevel AND pc.source_level_code = p.current_level_code AND (pc.education_condition = COALESCE(p.education_level, 'BACHELOR') OR pc.education_condition = 'ANY') AND pc.is_active = 1
       - WHERE p.current_level_code IN (source levels from step 1) AND p.is_active = 1
       - Use DATE_ADD(p.current_level_start_date, INTERVAL CAST(pc.min_years AS UNSIGNED) YEAR) for qualification_date (per Pitfall 4 — handles leap years)
       - Use DATEDIFF(qualification_date, CURDATE()) for remaining_days
       - CASE for status: NULL level_code or level_start_date => 'check_data', remaining_days <= 0 => 'qualified', else 'not_yet' (per D-10)
       - SELECT also: p.personnel_id, CONCAT(p.first_name, ' ', p.last_name) AS full_name, pos.position_name AS current_position, p.current_level_code, p.current_level_start_date, COALESCE(p.education_level, 'BACHELOR') AS education_level, pc.min_years, o.org_name AS department

    3. Apply search filter (per D-09): if $search not empty, add `AND (p.first_name LIKE ? OR p.last_name LIKE ? OR pos.position_name LIKE ?)` with `%{$search}%`

    4. ORDER BY remaining_days ASC (qualified first, then by nearest qualification date)

    5. Apply LIMIT/OFFSET using intval() interpolation (per Pitfall 3, same as civil-servants pattern): `LIMIT {$limit} OFFSET {$offset}`

    6. Run a separate count query (same WHERE, no LIMIT) for pagination total

    7. Run summary count queries:
       - Count WHERE status = 'qualified' (remaining_days <= 0 AND level data not null)
       - Count WHERE status = 'not_yet'
       - Count WHERE status = 'check_data'
       OR compute these from the full result set in PHP (simpler for v1 — just iterate the count query results)

    8. For each row, add formatted fields using helpers:
       - `qualification_date_thai` = formatThaiDate(qualification_date)
       - `level_start_date_thai` = formatThaiDate(current_level_start_date)
       - `current_level_name` = getLevelName(current_level_code)

    9. Return array with structure:
       ```php
       [
           'success' => true,
           'data' => $rows,
           'summary' => ['total' => $total, 'qualified' => $qualCount, 'not_yet' => $notYetCount, 'check_data' => $checkCount],
           'pagination' => ['total' => $total, 'limit' => $limit, 'offset' => $offset, 'has_more' => ($offset + $limit) < $total]
       ]
       ```
