**Conversion rules (same as Plan 01 Task 2):**
- `BIGSERIAL PRIMARY KEY` -> `BIGINT AUTO_INCREMENT PRIMARY KEY`
- `BOOLEAN DEFAULT FALSE/TRUE` -> `TINYINT(1) DEFAULT 0/1`
- `col BIGINT REFERENCES table(pk)` -> `col BIGINT` + `FOREIGN KEY (col) REFERENCES table(pk)`
- `DEFAULT CURRENT_DATE` -> `DEFAULT (CURDATE())`
- `TIMESTAMP DEFAULT CURRENT_TIMESTAMP` on updated_at -> add `ON UPDATE CURRENT_TIMESTAMP`
- `CREATE OR REPLACE VIEW` -> `DROP VIEW IF EXISTS ...; CREATE VIEW ...`
- `string1 || ' ' || string2` -> `CONCAT(string1, ' ', string2)`
- `(pe.end_date - CURRENT_DATE)` -> `DATEDIFF(pe.end_date, CURDATE())`
- `ps.is_active` (boolean test in WHERE) -> `ps.is_active = 1`
- Skip `COMMENT ON TABLE/VIEW` and all `INSERT INTO notification_config` from Part 8 (deferred per CONTEXT.md)
- Add `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci` to every CREATE TABLE

**Critical: probation_task_progress FK handling (per Research pitfall 7):**
- `training_participant_id` -> plain `BIGINT` column, NO FOREIGN KEY (training_participant table not created, too deep a dependency chain)
- `elearning_completion_id` -> plain `BIGINT` column, NO FOREIGN KEY

**View (after all tables):**

```sql
DROP VIEW IF EXISTS vw_probation_dashboard;
CREATE VIEW vw_probation_dashboard AS
SELECT
    pe.enrollment_id,
    p.personnel_id,
    p.citizen_id,
    CONCAT(p.first_name, ' ', p.last_name) AS full_name,
    p.hire_date,
    pe.start_date AS probation_start,
    pe.end_date AS probation_end,
    DATEDIFF(pe.end_date, CURDATE()) AS remaining_days,
    pe.overall_status,
    (SELECT COUNT(*) FROM probation_task_progress tp
     WHERE tp.enrollment_id = pe.enrollment_id) AS total_tasks,
    (SELECT COUNT(*) FROM probation_task_progress tp
     WHERE tp.enrollment_id = pe.enrollment_id AND tp.status = 'COMPLETED') AS completed_tasks,
    (SELECT COUNT(*) FROM probation_task_progress tp
     WHERE tp.enrollment_id = pe.enrollment_id AND tp.status = 'OVERDUE') AS overdue_tasks,
    (SELECT CONCAT(p2.first_name, ' ', p2.last_name)
     FROM probation_stakeholder ps
     JOIN personnel p2 ON ps.personnel_id = p2.personnel_id
     WHERE ps.enrollment_id = pe.enrollment_id AND ps.role_type = 'MENTOR' AND ps.is_active = 1
     LIMIT 1) AS mentor_name,
    (SELECT CONCAT(p2.first_name, ' ', p2.last_name)
     FROM probation_stakeholder ps
     JOIN personnel p2 ON ps.personnel_id = p2.personnel_id
     WHERE ps.enrollment_id = pe.enrollment_id AND ps.role_type = 'SUPERVISOR' AND ps.is_active = 1
     LIMIT 1) AS supervisor_name,
    (SELECT CONCAT(p2.first_name, ' ', p2.last_name)
     FROM probation_stakeholder ps
     JOIN personnel p2 ON ps.personnel_id = p2.personnel_id
     WHERE ps.enrollment_id = pe.enrollment_id AND ps.role_type = 'DIRECTOR' AND ps.is_active = 1
     LIMIT 1) AS director_name,
    o.org_name AS department,
    pos.position_name
FROM probation_enrollment pe
JOIN personnel p ON pe.personnel_id = p.personnel_id
LEFT JOIN organization o ON p.current_org_id = o.org_id
LEFT JOIN position pos ON p.current_position_id = pos.position_id
WHERE pe.overall_status = 'IN_PROGRESS';
```

Note: `ps.is_active` in WHERE must be `ps.is_active = 1` (not just `ps.is_active` which works in PostgreSQL but is ambiguous in MySQL views).
