- `BIGSERIAL PRIMARY KEY` -> `BIGINT AUTO_INCREMENT PRIMARY KEY`
- `BOOLEAN DEFAULT FALSE` -> `TINYINT(1) DEFAULT 0`

- `CREATE OR REPLACE VIEW` -> `DROP VIEW IF EXISTS ...; CREATE VIEW ...`
- Date arithmetic `(date1 - date2)` -> `DATEDIFF(date1, date2)` -- DATEDIFF argument order: first arg minus second
- `CASE WHEN x THEN TRUE ELSE FALSE END` -> `CASE WHEN x THEN 1 ELSE 0 END`
- Skip all `COMMENT ON TABLE/VIEW` statements (use inline SQL `--` comments instead)
- Add `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci` to every CREATE TABLE

**Views to create (after all tables):**

`vw_job_series_tenure`:
```sql
DROP VIEW IF EXISTS vw_job_series_tenure;
CREATE VIEW vw_job_series_tenure AS
SELECT
    pph.personnel_id,
    COALESCE(pph.job_series_name, p_pos.position_name) AS target_job_series,
    pph.effective_date AS tenure_start_date,
    COALESCE(pph.end_date, CURDATE()) AS tenure_end_date,
    DATEDIFF(COALESCE(pph.end_date, CURDATE()), pph.effective_date) AS total_days,
    CASE WHEN pph.end_date IS NULL THEN 1 ELSE 0 END AS is_current
FROM personnel_position_history pph
LEFT JOIN position p_pos ON pph.position_id = p_pos.position_id;
```

`vw_executive_tenure`:
```sql
DROP VIEW IF EXISTS vw_executive_tenure;
CREATE VIEW vw_executive_tenure AS
SELECT
    pph.personnel_id,
    COALESCE(pph.job_series_name, p_pos.position_name) AS position_name,
    pph.position_level,
    pph.effective_date AS start_date,
    COALESCE(pph.end_date, CURDATE()) AS end_date,
    DATEDIFF(COALESCE(pph.end_date, CURDATE()), pph.effective_date) AS total_days,
    CASE WHEN pph.end_date IS NULL THEN 1 ELSE 0 END AS is_current
FROM personnel_position_history pph
LEFT JOIN position p_pos ON pph.position_id = p_pos.position_id
WHERE pph.position_level IN ('M1','M2','S1','S2');
```

Preserve all column definitions, data types, and constraints from the PostgreSQL source. Include Thai inline comments (`--`) from the source schema to preserve business context.
